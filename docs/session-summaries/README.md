How to add a new session summary
-------------------------------

1. Create a new Markdown file named `SESSION_SUMMARY_YYYY-MM-DD.md` using the session date.
2. Add a short one-line description at the top and a summary of code changes, tests, and next steps.
3. Add a line to this `README.md` index (alphabetical or chronological order) linking the new file.

Suggested fields inside each session summary
-------------------------------------------

- Date & participants
- Goals and accepted criteria
- Code changes (files and purpose)
- Tests added/changed and quick results
- Live diagnostics and steps taken
- Remaining action items / recommended next steps

Why keep these
---------------

Session summaries make it easy to track what was changed, why it was changed, and the status of follow-ups. They are useful for audits, onboarding, and faster debugging.
